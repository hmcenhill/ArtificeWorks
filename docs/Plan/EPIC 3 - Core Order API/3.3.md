## Submit order endpoint

**Labels:** api, backend
**Milestone:** M2

## Summary

Implement `POST /orders/{id}/submit` to move a valid order into processing.

## Why

Submission is the bridge between synchronous API behavior and asynchronous workflow.

## Tasks

- [ ] Validate order can be submitted
- [ ] Transition order to submitted/payment-pending state
- [ ] Save state change
- [ ] Prepare hook for event publication
- [ ] Add integration tests

## Acceptance Criteria

- [ ] Valid draft order can be submitted
- [ ] Invalid state submission is rejected
- [ ] Order state updates are persisted
- [ ] Integration test covers success and invalid submit

## Notes

Start with clear state behavior even before messaging is fully wired.
