## [EPIC] Core order API

**Labels:** epic, api, backend
**Milestone:** M2

## Summary

Expose REST endpoints for creating, reading, submitting, and cancelling orders.

## Why

The API is the entry point to the order workflow and the easiest way to demonstrate the system.

## Acceptance Criteria

- [ ] Create endpoint exists
- [ ] Get-by-id endpoint exists
- [ ] Submit endpoint exists
- [ ] Cancel endpoint exists
- [ ] Core API behavior is covered by integration tests
