## Integration test coverage

**Labels:** api, testing
**Milestone:** M2

## Summary

Close the integration-test gaps across the full API surface: lifecycle commands, cancellation, error contract, and product endpoints.

## Why

Epic 3 is the last synchronous epic — this suite is the safety net the async pivot (Epic 4+) will constantly lean on.

## Tasks

- [x] Happy-path coverage for every endpoint (products and work orders)
- [x] A full lifecycle walk: create → advance ×N → completed, asserting state history at each step
- [x] Hold/release and cancel flows, including rejections
- [x] Error-contract assertions per 3.3 (every `ProblemCodes` value is now asserted somewhere)
- [x] Concurrent **create** smoke check (no lost writes, no duplicated history) — see note on concurrent *advance* below

## Acceptance Criteria

- [x] Every endpoint has at least one success and one failure test
- [x] The full-lifecycle test passes against real Postgres (Testcontainers)
- [x] CI runs the suite on every push (existing GitHub Actions workflow runs `dotnet test`)

## Notes / decisions

- **New tests (9):** `ProductApiTests` (create success, duplicate→409, get success, get missing→404) + additions to
  `WorkOrderApiTests` (history-missing→404, hold-already-held→409, advance-while-held→409, full-lifecycle walk,
  concurrent-create smoke check). Total suite: 45 unit + 23 integration = 68 green.
- Extracted the ProblemDetails `code` reader into a shared `ApiTestExtensions.ReadProblemCodeAsync` so both API test
  classes assert the wire contract the same way.
- **Concurrent *advance* deliberately not tested.** `WorkOrder` has no optimistic-concurrency token (no rowversion /
  Postgres `xmin` mapping). Two simultaneous advances on the *same* order each load the same state, both append history,
  and both save — producing duplicate history / a lost update with no exception. Writing a "no duplicate history" assertion
  there would encode a failing/flaky test against current behaviour. The concurrent-*create* check is safe (distinct
  aggregates, no shared row) and is the meaningful smoke test today. Concurrency control on the aggregate is deferred —
  the Epic 4+ async pivot moves lifecycle transitions onto a message-driven path and will revisit this. Flagged in HANDOFF.
