## Configure RabbitMQ publisher abstraction

**Labels:** messaging, infra
**Milestone:** M3

## Summary

Create message publishing infrastructure and a simple abstraction for publishing events.

## Why

Publishing should be easy to test and easy to swap or evolve later if needed.

## Tasks

- [ ] Add RabbitMQ client configuration
- [ ] Define publisher abstraction/interface
- [ ] Configure exchange and routing basics
- [ ] Include correlation metadata support
- [ ] Add basic verification test or manual validation

## Acceptance Criteria

- [ ] App can publish a typed event
- [ ] Exchange/queue configuration is in place
- [ ] Correlation ID is included in message metadata
- [ ] Local publish flow works reliably

## Notes
