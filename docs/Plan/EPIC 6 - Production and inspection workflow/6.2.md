## Inspection stage and per-unit verdicts

**Labels:** messaging, backend, domain, api

## Summary

The second consumer: on `work-order.production-completed`, advance InProcess → Inspection and issue a pass/fail verdict for each newly built unit. Verdicts come from a configurable auto-inspector (failure rate defaults to `0.0`) or from an API call. This story delivers the **all-pass path** — full quantity passes, order advances to Delivery, `work-order.inspection-passed` published for Epic 7. Failures are recorded but their consequence is 6.3.

## Why

Inspection is where the domain gets its judgment call, and where the demo gets its interactive moment: the hybrid model says visitors make key decisions, and "does this unit pass?" is the most natural one in the whole pipeline. Building the verdict endpoint here — rather than letting Epic 12 bolt on a back door later — means failure injection rides the same path a real inspector would.

## Tasks

- [x] `StockKeepingUnit` verdict behaviour: `Pass()` and `Scrap(reason)` moving `Built` → `Passed` / `Scrapped`, stamping `InspectedUtc`, rejecting a second verdict on an already-inspected unit
- [x] `Application/Inspection/InspectionService`: load the order and its `Built` units, obtain a verdict per unit, apply it, note history, and publish the outcome
- [x] Verdict source behind an interface (`IVerdictSource` or similar) with a **configurable random implementation** — `Inspection:FailureRate`, default `0.0`, plus an optional seed so tests are deterministic. Keeping it behind an interface is what lets Epic 10 swap in the simulation engine without touching the workflow
- [x] `POST /work-orders/{id}/inspections` recording a manual verdict for a serial number (`passed`, `reason`), following the established ProblemDetails contract — unknown serial, wrong order, already inspected, and order-not-in-Inspection all get stable `code`s
- [x] Worker handler `ProductionCompletedHandler`, thin adapter as before
- [x] New event contract `InspectionPassed` (`work-order.inspection-passed`) — the full ordered quantity has passed — carrying the passing serial numbers. Nothing binds it yet; Epic 7's shipping consumer is its first subscriber
- [x] Expose inspection state on the read side: units and their verdicts on the work-order response, so a failure is visible without reading the log
- [x] Tests: unit tests for verdict application and the double-verdict guard; a deterministic-seed test proving the failure rate is honoured; integration test for the all-pass chain scheduled → picked → produced → inspected → Delivery, observing `InspectionPassed`

## Acceptance Criteria

- [x] Inspection produces an explicit pass/fail outcome per serialized unit, and a failure carries a reason
- [x] With `FailureRate = 0.0`, a work order runs unattended from creation to Delivery with no human action
- [x] A verdict can also be recorded through the API, and the two paths write the same domain state
- [x] A unit cannot be inspected twice
- [x] `InspectionPassed` fires only when the full ordered quantity has passed
- [x] Every transition is in the state history, and verdicts are visible on the API

## Decisions (to confirm at story start)

- **Auto-verdict plus API override**, settled at grooming. The auto-inspector keeps the demo alive; the endpoint is the visitor's decision moment and Epic 12's injection point.
- **Verdicts are recorded per unit, immediately** — not batched into a single order-level verdict. The order-level outcome is derived by counting passing units against `OrderItemQty`.
- **This story does not implement the scrap consequence.** A failing unit is marked `Scrapped` with its reason, but the order does not yet rebuild — 6.3 owns the loop. Until then a failing order simply sits in Inspection short of quantity, which is an acceptable half-step because the default failure rate is zero.
- **The manual endpoint does not require the order to be idle.** The auto-inspector and a human could in principle both verdict the same unit; the `already inspected` guard is the resolution, and it should be a clean 409 rather than a race to think hard about.

## Notes

Resist making the verdict source clever. It is a coin flip behind an interface. The moment it wants to know about component quality, operator skill, or machine wear, it has become Epic 10 and belongs there.
