## Add API idempotency keys for order submission

**Labels:** reliability, api
**Milestone:** M4

## Summary

Support idempotency keys so duplicate submit requests do not create duplicate submission effects.

## Why

Clients retry requests in the real world; the API should be resilient to this.

## Tasks

- [ ] Define idempotency key contract
- [ ] Persist idempotency records
- [ ] Return stable response for repeated requests
- [ ] Document expiry/retention strategy
- [ ] Add duplicate-submit tests

## Acceptance Criteria

- [ ] Same-key duplicate requests create exactly one logical submission result
- [ ] Repeated request returns stable outcome
- [ ] Idempotency behavior is documented
- [ ] Tests prove duplicate submissions are safe

## Notes

This can use Redis or Postgres depending on what stays simplest.
