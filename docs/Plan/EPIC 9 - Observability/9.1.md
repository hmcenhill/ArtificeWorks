## Add structured logging with correlation IDs

**Labels:** observability, backend
**Milestone:** M5

## Summary

Implement structured logging across API and workers with correlation IDs.

## Why

Without correlation, async workflows are much harder to inspect and explain.

## Tasks

- [ ] Generate/propagate correlation ID from request
- [ ] Include order ID and correlation ID in log context
- [ ] Propagate correlation into published messages
- [ ] Preserve correlation in worker logs
- [ ] Verify traceability manually

## Acceptance Criteria

- [ ] Every request has a correlation ID
- [ ] Worker logs include propagated correlation ID
- [ ] Logs can be filtered by order ID or correlation ID
- [ ] One order can be followed across multiple stages

## Notes

This should be visible during demos.
